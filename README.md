#  화성시 행사 관리 및 실시간 VIP 통제 서비스

> **스프링 부트와 웹소켓, 카카오 API를 활용한 실시간 현장 참석자 대시보드 및 알림 시스템**

---

##  주요 기능
* **QR 현장 인증 및 승인 (`qr.html`)**: 참석자가 이름과 연락처를 입력하여 현장 출석을 인증합니다.
* **실시간 관리자 대시보드 (`ex.html`)**: 웹소켓(WebSocket/SockJS)을 통해 페이지 새로고침 없이 실시간으로 출석 현황을 동기화합니다.
* **카카오톡 VIP 알림 연동 (`ChatController.java`)**: 웹소켓 수신 데이터 중 VIP 등급이 감지되면, 카카오 REST API를 통해 최고 관리자에게 즉시 실시간 카카오톡 알림을 전송합니다.

---

##  기술 스택 (Tech Stack)
* **Backend**: Java, Spring Boot
* **Frontend**: HTML5, CSS3, Thymeleaf, JavaScript, Bootstrap 5
* **Real-time Communication**: WebSocket, SockJS
* **Data Processing & Library**: Jackson (`ObjectMapper`), kakao API

---
##  시스템 아키텍처 흐름 
*행사 관리*
1. **최고 관리자 로그인 및 세션 생성**
   * 관리자가 `login.html`을 통해 로그인을 수행하면, `ChatController.java`에서 `manager.json`을 검증한 뒤 인증 성공 시 해당 관리자의 이름, 역할, 그리고 연락처(`010-5751-9060`)를 HTTP 세션(`HttpSession`)에 안전하게 넣습니다.

2. **`ChatWebSocketHandler.java`를 통해 대시보드로 데이터 실시간 브로드캐스팅**
   * 현장 데스크에서 참석자 인증이 완료되어 웹소켓 신호가 들어오면, 서버 백엔드 핸들러의 `@Override handleTextMessage()` 메서드가 동작하여 연결된 모든 관리자 대시보드 창(`ex.html`)으로 유저 데이터 JSON 문자열을 실시간 토스합니다.

3. **`ex.html`에서 데이터 수신 후 VIP 등급 판별**
   * 대시보드 창의 `sock.onmessage` 리스너 함수가 데이터를 캐치하여 객체로 변환한 뒤, `if (data.grade === 'VIP')` 조건문을 가동해 VIP 여부를 실시간 판별합니다.

4. **백엔드 `/api/notify-manager` 주소로 비동기 POST 요청**
   * VIP임이 확인되는 순간, 대시보드 자바스크립트 내부에서 `fetch()`를 엔진 삼아 name과 grade 파라미터를 실어 자바 서버로 비동기 요청을 전송합니다.

5. **`ChatController.java`에서 최고 관리자 세션 검증 후 카카오톡 공식 API 서버 타격 및 발송 완료**
   * `@PostMapping("/api/notify-manager")` 메서드가 요청을 받아 1단계에서 세션에 저장해 둔 관리자 번호가 최고 관리자(`01057519060`)와 일치하는지 2중 필터링합니다.
   * 검증 통과 시 `RestTemplate`에 유효한 카카오 액세스 토큰을 얹고, 메시지 내부 링크 도메인을 카카오 공식 주소로 고정하여 플랫폼 도메인 검증 필터를 우회(Bypass)시킨 후 최종 발송을 완료합니다.

*행사 기획*
1. **행사 목표 수립 및 요구사항 정의 (Event Planning)**
   * 화성시 주관 대규모 행사의 특성상 귀빈(VIP)의 참석률이 높고 현장 혼잡도가 예상됨에 따라, 아날로그식 수기 명부를 폐기하고 실시간 동기화 시스템 및 자동화된 통제 알림 인프라 구축을 최우선 목표로 기획했습니다.

2. **관계자 및 참석자 데이터 규격화 설계 (Data Modeling)**
   * 개인정보 유출을 방지하고 시스템 독립성을 확보하기 위해 로컬 임시 DB 역할을 할 구조적 스키마를 설계했습니다.
   * 행사 총괄자 계정 및 보안 연락처를 담은 `manager.json`과 당일 참석 대상자의 등급(VIP/일반) 및 고유 식별값을 지정한 `attendance.json` 데이터를 구축했습니다.

3. **현장 보안 및 통제 필터 가이드라인 수립 (Security Policy)**
   * 현장 인증 시 유출 사고를 막기 위해 프론트엔드와 백엔드 통신 간 모든 연락처 데이터에서 하이픈(-)을 강제로 제거하고 문자열을 화이트리스트 기반으로 정제(Trim)하는 보안 필터 정책을 수립했습니다.
   * 악의적인 무단 입장이나 반복적인 인증 실패를 감지하기 위한 연속 3회 실패 시 대시보드 경고 알림(isWarning) 발송 규칙을 설계했습니다.

4. **네트워크 및 하드웨어 인프라 배치 기획 (Infrastructure Deployment)**
   * 현장 안내 데스크의 QR 스캔 단말기(Client A)와 총괄 책임자의 모니터링 대시보드 PC(Client B) 간의 무선 네트워크 망을 설계했습니다.
   * 브라우저의 무분별한 리로딩(새로고침)으로 인한 트래픽 과부하를 방지하기 위해, 웹소켓 영구 커넥션을 활용한 실시간 이벤트 트리거형 디스플레이 배치를 확정했습니다.
